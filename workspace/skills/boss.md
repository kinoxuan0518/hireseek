# BOSS 直聘兜底执行协议

> 这不是 canonical BOSS skill。真实运行协议应来自本机技能库：
> `HIRESEEK_SKILL_HOME/bosszhibin-auto-recruiter/SKILL.md`。
> 本文件只保留最低限度兜底，避免外部技能缺失时 HireSeek 完全不能跑。

## 不变量

1. 默认接管用户已登录的真实 Chrome 页面，禁止新开无头浏览器或临时登录会话。
2. 候选人处理前必须先使用 BOSS 页面筛选项收窄人群，再逐个精筛。
3. 触发每日主动沟通上限时立即停止，不再点击、滚动或切页。
4. 出现频率告警时等待 10-30 秒后再继续。
5. 同一候选人同一轮最多打一次招呼。
6. 每打完一个招呼，必须立即调用 `record_contacted` 登记候选人结构化信息。

## 就地接管模式

当用户使用 `hireseek run boss --here` 时，含义只是：复用用户当前真实 Chrome 页面，不新开浏览器、不创建新登录态、不切换 profile。

- 禁止 `goto` 直接跳深链，禁止新开页面或新登录态。
- 允许通过当前页面内可见入口完成站内流转，例如职位下拉、职位列表、职位管理、推荐牛人入口。
- 如果当前职位不是目标岗位，应优先站内切到目标岗位，再处理推荐候选人。
- 如果找不到目标岗位、登录失效或风控阻断，再停止并说明需要用户处理什么。

## 筛选优先级

先用页面筛选面板表达可映射规则：

- 经验：按岗位要求选择，默认覆盖 1-3 年及相邻可接受区间。
- 学历：本科及以上。
- 学校：985、国内外名校等可映射标签。
- 关键词：岗位相关技术词，如 Agent、LLM、大模型、RAG、Python、平台等。

无法用平台筛选表达的规则，再用候选人卡片和详情文本做二次判断。

主要跳过原因: <简短描述>
