# 脉脉招聘自动化执行协议

版本：3.1  
语言：中文  
执行方式：浏览器自动化

## 0. 执行边界

- 仅在用户明确要求脉脉招聘相关操作时执行。
- 仅使用已登录的脉脉企业招聘页面（`maimai.cn/ent`）。
- 把筛选条件当作硬性条件处理；不把硬性条件降级为“加分项”。
- 始终保持人机协作：首批与每 5 人批次给用户确认。

## 1. 启动前检查

按顺序执行：

1. 确认浏览器会话可用且已登录脉脉招聘端。
2. 进入人才搜索页：`https://maimai.cn/ent/v41/recruit/talents?tab=1`。
3. 检查缓存目录（`${MAIMAI_CACHE_DIR:-~/.hireclaw/maimai_cache/}`）。
- 路径通过 `MAIMAI_CACHE_DIR` 环境变量配置；若目录不存在则自动创建。
- 若环境不可写，改用当前可写目录并告知用户。
4. 检查岗位配置缓存和触达历史缓存。

缓存结构见：`references/cache-schema.md`

## 2. 读取或创建岗位配置

- 命中缓存时：展示当前岗位配置，要求用户“确认/调整”。
- 未命中缓存时：收集岗位名、城市、学历、年限、学校层级、目标公司类型、技术方向。
- 生成“搜索轮次矩阵”：每轮 = `关键词 × 公司筛选组合`。
- 配置保存后再执行搜索。

关键词和轮次设计见：`references/search-playbook.md`

## 3. 分轮次搜索执行

对每个轮次执行以下流程：

1. 设置关键词。
2. 设置筛选器：毕业学校、就职公司、城市、学历、工作年限。
3. 提交搜索并等待结果稳定。
4. 记录轮次名、筛选条件、结果数量。

### 3.1 轮内筛选规则

在平台筛选完成后，继续执行人工硬筛：

- 按钮状态：
- `沟通` = 已触达，跳过。
- `立即沟通` = 可触达，继续。
- 专业检查：教育经历明显非计算机相关则跳过。
- 去重检查：命中历史缓存或本轮内存去重则跳过。

### 3.2 评分与排序

- 使用统一 100 分模型排序候选人优先级。
- 评分仅用于排序，不用于替代硬性过滤。
- 输出时展示分项构成（公司/学校/活跃/意愿）。

评分细则见：`references/search-playbook.md`

## 4. 候选人确认与触达

### 4.1 候选人确认

每批输出以下信息并等待用户指令：

- 通过筛选人数与排序列表。
- 跳过人数与跳过原因。
- 操作选项：全部触达 / 选择性触达 / 跳过本批。

### 4.2 消息生成

为每位候选人生成专属消息并满足：

1. 使用候选人真实信息（至少公司名或技术方向）。
2. 长度建议 100-180 字，严格小于 300 字。
3. 不复制模板，不群发口吻，不夸张吹捧。
4. 语气平等、直接、自然。

消息策略见：`references/outreach-playbook.md`

### 4.3 发送动作

在“招聘立即沟通”弹窗中依次执行：

1. 选择对应职位。
2. 粘贴个性化消息。
3. 开启附带职位与智能索要（若可见且默认开启则保持开启）。
4. 点击“发送后留在此页”。
5. 验证按钮由“立即沟通”变为“沟通”。
6. 写入触达历史缓存。

## 5. 轮次切换与终止条件

### 5.1 当前轮次终止

满足任一条件即结束当前轮次：

- 连续 2 次滚动无新候选人。
- 连续 3 批无通过筛选候选人。
- 用户要求停止当前轮次。

### 5.2 全局终止

满足任一条件即结束任务：

- 所有轮次完成。
- 用户主动结束。
- 平台触达上限或权限限制触发。
- 已达到用户要求触达数量。

## 6. 报告与缓存更新

结束后输出执行报告，至少包含：

1. 基本信息：岗位、执行起止时间。
2. 分轮次统计：浏览、通过、触达。
3. 触达明细：姓名、公司、评分、轮次。
4. 跳过原因统计：专业不符、已触达、其他。
5. 下次优化建议：轮次增删、关键词收敛或扩展。

然后更新：

- `job_configs.json`（轮次和规则）
- `outreach_history.json`（触达明细和日统计）

## 7. 异常处理

- 页面结构变化：截图并重新定位关键控件。
- 搜索无结果：建议放宽条件，记录优化建议。
- 无法“立即沟通”：判断为上限或权限，终止并汇报。
- 验证码：暂停自动化并通知用户手动处理。
- 弹窗失败：最多重试 3 次，失败后跳过该候选人。
- 公司名称不匹配：尝试公司别名或英文名变体。

平台控件细节见：`references/platform-ui-reference.md`

## 8. 执行质量标准

- 全程幂等：同一候选人同一天不重复触达。
- 过程可复盘：每轮都能回溯筛选条件和结果。
- 消息可解释：每条都能对应候选人背景依据。
- 退出可恢复：中断后可基于缓存继续下一轮。
