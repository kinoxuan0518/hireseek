# Search Playbook（搜索策略手册）

脉脉搜索的关键词策略与搜索矩阵设计规范。

---

## 搜索矩阵设计原则

每个岗位的搜索由多个"轮次"组成，每轮 = **关键词 × 公司类型筛选**。

```
轮次 = 关键词组 × 公司筛选
```

- **宽进严出**：关键词宁可宽泛，靠后续脚本精筛
- **去重优先**：每轮结束后标记已触达，下轮自动跳过
- **优先级排序**：高匹配度轮次在前，泛化轮次在后

---

## 轮次矩阵示例（技术岗位）

> 以下为示例结构，根据实际 JD 自行填写关键词。

| 轮次 | 关键词 | 公司筛选 | 说明 |
|------|--------|----------|------|
| r1 | 核心技能词A、技能词B | 目标公司列表 | 精准高优轮次 |
| r2 | 核心技能词A | 大厂泛筛 | 扩大范围 |
| r3 | 泛化技能词 | 不限公司 | 兜底轮次 |
| r4（可选）| 应届关键词 | 顶校筛选 | 应届补充轮 |

## 关键词策略

### 技术岗位关键词三层结构

```
核心层（必须命中，1-2 个）
  └── 岗位核心技术方向，如：机器学习、前端开发

扩展层（命中加分，2-4 个）
  └── 细分方向，如：推荐系统、微服务、性能优化

泛化层（兜底，1-2 个）
  └── 通用词，如：算法、工程师
```

### 非技术岗位关键词结构

```
行业词（1-2 个）
  └── 所在行业或业务方向

职能词（1-2 个）
  └── 岗位职能关键词

经验词（1 个，可选）
  └── 特定业务经验，如：0-1 产品、B端产品
```

## 脉脉平台筛选器映射

| 搜索意图 | 脉脉平台筛选项 | 备注 |
|----------|---------------|------|
| 学历要求 | `学历` → 本科/硕士/博士 | 支持多选 |
| 工作年限 | `工作年限` → 1-3年/3-5年/5-10年 | 支持多选 |
| 院校背景 | `毕业学校` → 985/211/海外名校 | 平台支持标签筛选 |
| 就职公司 | `就职公司` → 输入公司名 | 支持多个，逗号分隔 |
| 所在城市 | `城市` → 选择目标城市 | 支持多城市 |

## 评分细则（脉脉版）

总分 100 分，与 BOSS 直聘共享基础维度：

| 维度 | 满分 | 说明 |
|------|------|------|
| 学校层级（共享） | 25分 | 按院校分级，详见 `global_rule_defaults.school_policy` |
| 公司匹配（共享） | 30分 | 按目标公司匹配度，详见职位配置 |
| 活跃度（触达专用）| 25分 | 最近活跃时间评分 |
| 求职意愿（触达专用）| 20分 | 求职状态评分 |

> 具体分段由各公司根据自身偏好在 `job_configs.json` 中配置。

---

## 轮次终止与切换规则

- 连续 2 次滚动无新候选人 → 当前轮次结束
- 连续 3 批（每批 10 人）无通过筛选 → 当前轮次结束
- 所有轮次完成 → 任务结束
- 用户主动结束 → 立即停止，输出进度快照
