# Source Map

用于告诉 `talent-sourcing` 在不同任务下优先看哪些 source，而不是一上来全网乱搜。

## 1. 常规渠道寻人

- 优先 source：脉脉、LinkedIn、BOSS / 其他招聘平台
- 辅助验证：GitHub、Hugging Face、Google Scholar / arXiv

## 2. 冰山以下人才挖掘

- 优先 source：GitHub、Google Scholar、arXiv、Hugging Face、个人主页 / 博客、项目主页 / contributor 页面、实验室主页 / 公司团队页

输出重点：

- 为什么属于目标画像
- 证据来自哪个 source
- 适合通过什么渠道触达

## 3. 指定人物调研

- 优先 source：LinkedIn、GitHub、Google Scholar / arXiv、公开网页搜索、项目页 / 团队页

必须回答：

- 身份是否能被交叉验证
- 方向是否真的匹配
- 是硬证据还是推断
- 是否值得进入触达流程

## 4. Builder / Operator 型人才映射

- 优先 source：LinkedIn、GitHub、公司 team page、产品发布页 / launch page / changelog、个人博客 / 访谈 / conference page

输出重点：

- 共同 archetype 是什么
- 哪些是产品或项目硬证据
- 哪些公司/项目是高密度来源
- 下一步更适合从哪个渠道继续扩展或触达
