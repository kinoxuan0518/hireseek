# Platform UI Reference（平台控件参考）

记录各招聘平台关键 UI 元素的选择器与交互说明，供浏览器自动化兜底使用。

> **注意**：平台会不定期更新 UI，选择器可能失效。Agent 应优先通过视觉识别定位元素，本文件作为参考。

---

## 脉脉招聘端（maimai.cn/ent）

### 人才搜索页
- 入口：`https://maimai.cn/ent/v41/recruit/talents?tab=1`
- 关键词输入框：顶部搜索栏
- 筛选器：搜索结果页左侧或顶部筛选面板

### 候选人卡片
- 候选人列表：滚动区域，每张卡片含姓名/职位/公司/学校
- **立即沟通** 按钮：蓝色主按钮，表示可触达
- **沟通** 按钮：灰色或浅色，表示已触达，跳过

### 发送消息弹窗
操作顺序：
1. 点击"立即沟通"
2. 在弹窗中选择对应职位
3. 清空并粘贴个性化消息
4. 确认"附带职位"和"智能索要"开关状态
5. 点击"发送后留在此页"
6. 验证按钮变为"沟通"（表示成功）

### 常见异常
- 验证码：识别到验证码弹窗后立即暂停，等待用户手动处理
- 触达上限：出现权限提示或按钮不可用，停止当前任务

## BOSS 直聘企业端（zhipin.com）

### 推荐牛人页
- 入口：`https://www.zhipin.com/web/employer/talent/recommend`
- 职位切换：顶部或左侧职位下拉/列表
- 页签：推荐 / 最新 / 精选（`li.tab-item`）

### 候选人卡片
- 候选人列表：主内容区滚动列表
- **打招呼** 按钮：主要操作按钮
- **继续沟通** 按钮：表示已联系，跳过

### 筛选面板
操作顺序：
1. 点击"筛选"按钮
2. 若出现"应用上次筛选"提示，选择"取消"
3. 依次设置：经验要求 / 学历 / 院校 / 关键词 / 活跃时间
4. 点击"确定"
5. 验证筛选标签是否激活

### 打招呼流程
1. 点击"打招呼"
2. 等待弹窗出现
3. 发送消息（使用平台预设或自定义消息）
4. 验证按钮状态变为"继续沟通"

### 风控弹窗识别
| 弹窗文字 | 处理方式 |
|----------|----------|
| 今日主动沟通数已达上限 | 立即停止，进入总结 |
| 开聊太频繁 | 等待 10-30 秒后继续 |
| 操作太频繁 | 等待 10-30 秒后继续 |
| 账号未登录 | 暂停，提示用户登录 |

## LinkedIn 招聘端（linkedin.com/talent）

> LinkedIn 反爬较强，建议配合登录 Cookie 使用，并降低操作频率。

### 候选人搜索
- 入口：`https://www.linkedin.com/talent/hire`
- 关键词搜索：顶部搜索框
- 筛选：地区 / 职能 / 行业 / 公司规模

### 候选人卡片
- **Connect** / **Message** 按钮：主要触达操作
- **Pending** 状态：已发送连接请求，跳过

## 通用自动化注意事项

1. **操作节奏**：相邻操作间隔 1-3 秒，避免触发风控
2. **滚动加载**：使用渐进滚动，等待新内容加载完成再操作
3. **选择器优先级**：视觉识别 > 稳定 CSS 选择器 > 文本匹配
4. **截图验证**：每次关键操作后截图确认状态
5. **幂等保护**：同一候选人同一天不重复操作
